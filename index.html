The Cars Web API is based on MongoDB,Postman and URL/Requests. Data resources are accessed via standard HTTPS requests in UTF-8 format to an API endpoint. Where possible, Web API uses appropriate HTTP verbs for each action:

Method Action
Retrieves resources
Creates resources
Changes and/or replaces resources or collections
Deletes resources

To access the API, you'll need to authenticate using an API key. You can generate an API key by visiting the "API Keys" section of your account settings.

Cars API

Here's a list of the routes available in the API:

Method Endpoint Description
/cars Get a list of all the cars
Functions
sort: allows you to view price from high to low
Example: https://carapi-eyal.cyclic.app//cars?sort=price

Rate Limiting

The API is rate-limited to 100 requests per hour per IP address. If you exceed this limit, you'll receive a 429 "Too Many Requests" response.

Error Handling

The API uses standard HTTP status codes to indicate the success or failure of a request. Here are some of the most common status codes:

Status Code Description
200 OK - The request has succeeded. The client can read the result of the request in the body and the headers of the response.
201 Created - The request has been fulfilled and resulted in a new resource being created.
202 Accepted - The request has been accepted for processing, but the processing has not been completed.
204 No Content - The request has succeeded but returns no message body.
304 Not Modified.
400 Bad Request - The request could not be understood by the server due to malformed syntax. The message body will contain more information.
401 Unauthorized - The request requires user authentication or, if the request included authorization credentials, authorization has been refused for those credentials.
403 Forbidden - The server understood the request, but is refusing to fulfill it.
404 Not Found - The requested resource could not be found. This error can be due to a temporary or permanent condition.
429 Too Many Requests has been applied.
500 Internal Server Error. You should never receive this error because our clever coders catch them all … but if you are unlucky enough to get one, please report it to us through a comment at the bottom of this page.
502 Bad Gateway - The server was acting as a gateway or proxy and received an invalid response from the upstream server.
503 Service Unavailable - The server is currently unable to handle the request due to a temporary condition which will be alleviated after some delay. You can choose to resend the request again.
